# Voting (Coming Soon)

This sub-pattern is not built yet.

**Voting** runs the *same* task multiple times (different prompts, models, or seeds) and aggregates
the results by consensus — by majority vote, averaging, or a final judge. It trades extra LLM calls
for higher confidence on subjective or error-prone judgments.

Contrast with its sibling, [Sectioning](../1-sectioning/README.md), which runs *different*
sub-tasks in parallel and merges them.
